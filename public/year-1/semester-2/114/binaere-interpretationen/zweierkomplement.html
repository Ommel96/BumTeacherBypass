Modul 114 -- Codieren

Zweierkomplement

Referenzdokument 10.4.3 · Quelle: bbw, P. Rutschmann
1

Vorgehen: Zweierkomplement berechnen

Um das Zweierkomplement einer positiven Zahl zu berechnen (also die negative Darstellung), gehen Sie in zwei Schritten vor:

  1. Alle Bits invertieren (drehen): aus 0 wird 1, aus 1 wird 0
  2. 1 dazu addieren
Beispiel: Zweierkomplement von 1 1 = 00000001 Schritt 1: Bits invertieren 00000001 → 11111110 Schritt 2: 1 addieren 11111110 + 00000001 = 11111111 Ergebnis: -1 = 11111111
Merke: Negative Zahlen erkennt man daran, dass das höchstwertige Bit (Bit 7) den Wert 1 hat. Das Verfahren funktioniert auch umgekehrt: Zweierkomplement einer negativen Zahl ergibt die positive Zahl.
2

Beispiele

1 → -1
1 = 00000001 11111110 + 00000001 -1 = 11111111
7 → -7
7 = 00000111 11111000 + 00000001 -7 = 11111001
6 → -6
6 = 00000110 11111001 + 00000001 -6 = 11111010
15 → -15
15 = 00001111 11110000 + 00000001 -15 = 11110001
16 → -16
16 = 00010000 11101111 + 00000001 -16 = 11110000
57 → -57
57 = 00111001 11000110 + 00000001 -57 = 11000111
126 → -126
126 = 01111110 10000001 + 00000001 -126 = 10000010
127 → -127
127 = 01111111 10000000 + 00000001 -127 = 10000001

Umgekehrt: Negativ → Positiv

Beispiel: Zweierkomplement von -43 -43 = 11010101 Bits invertieren: 00101010 1 addieren: + 00000001 = 00101011 = 43

Binäre Addition

Bits werden Spalte für Spalte zusammengezählt. Überträge (Carry) werden berücksichtigt:

Beispiel: 1 + 3 = 4 00000001 (1) + 00000011 (3) ---------- 00000100 (4) Bit 0: 1+1 = 10 → 0 notieren, 1 Übertrag Bit 1: 0+1+1(Übertrag) = 10 → 0 notieren, 1 Übertrag Bit 2: 0+0+1(Übertrag) = 1 → 1 notieren

Addition mit mehreren Zahlen

Beispiel: 3+3+3+3+3+3 = 18 00000011 00000011 00000011 00000011 00000011 + 00000011 ---------- 00010010 (18) Bit 0: 6×1 = 6 = 110 → 0 notieren, 11 Übertrag Bit 1: 6×1 + 11 = 111 → 1 notieren, 11 Übertrag Bit 2: 6×0 + 11 = 10 → 0 notieren, 1 Übertrag Bit 3: 6×0 + 1 = 1 → 1 notieren Immer das letzte Bit des Resultats notieren!
4

Binäre Subtraktion

Beispiel: 6 - 1 = 5 00000110 (6) - 00000001 (1) ---------- 00000101 (5) Bit 0: 0-1 → Übertrag nötig, ergibt 1

Negatives Resultat

Beispiel: 3 - 5 = -2 00000011 (3) - 00000101 (5) ---------- 11111110 (-2) Die Überträge pflanzen sich durch alle höheren Bits fort. Das Ergebnis ist automatisch im Zweierkomplement!

Subtraktion als Addition des Zweierkomplements

Statt zu subtrahieren kann man das Zweierkomplement addieren:

Beispiel: 3 - 1 = 3 + (-1) 3 = 00000011 + (-1) = 11111111 ----------- 100000010 ^^^^^^^^ Das 9. Bit (Überlauf) wird ignoriert: = 00000010 (2)
Beispiel: 3 - (-1) = 3 + 1 = 4 Zweierkomplement von -1 (= 11111111) ist 1 (= 00000001) 00000011 (3) + 00000001 (1) ---------- 00000100 (4)
5

Binäre Multiplikation

Prinzip: Multiplikation mit einer Zweierpotenz entspricht dem Verschieben (Shift) aller Bits nach links.
• Multiplikation mit 2 = 1× links schieben
• Multiplikation mit 4 = 2× links schieben
• Multiplikation mit 8 = 3× links schieben
Beispiel: 9 × 2 und 9 × 4 9 = 00001001 9×2 = 00010010 (18) ← 1× links geschoben 9×4 = 00100100 (36) ← 2× links geschoben

Komplexere Multiplikation

Den zweiten Faktor in Zweierpotenzen zerlegen, dann die Teilresultate addieren:

Beispiel: 9 × 13 = 117 13 = 8 + 4 + 1 = 23 + 22 + 20 Also: 9×13 = 9×8 + 9×4 + 9×1 9×8 = 00001001 « 3 = 01001000 (72) 9×4 = 00001001 « 2 = 00100100 (36) 9×1 = 00001001 « 0 = 00001001 (9) ---------- 01110101 (117)
Beispiel: 7 × 11 = 77 11 = 8 + 2 + 1 = 23 + 21 + 20 7×8 = 00000111 « 3 = 00111000 (56) 7×2 = 00000111 « 1 = 00001110 (14) 7×1 = 00000111 « 0 = 00000111 (7) ---------- 01001101 (77)
6

Binäre Division

Prinzip: Division durch eine Zweierpotenz entspricht dem Verschieben (Shift) aller Bits nach rechts.
• Division durch 2 = 1× rechts schieben
• Division durch 4 = 2× rechts schieben
Beispiel: 16 / 4 = 4 4 = 22 → 2× rechts schieben 16 = 00010000 00010000 » 2 = 00000100 (4)

Schriftliche Division

Für beliebige Divisoren wird schriftlich dividiert, genau wie im Dezimalsystem:

Beispiel: 107 / 7 = 15 Rest 2 Dezimal: 107 / 7 = 15 Rest 2 -0 10 - 7 37 -35 2 Binär: 01101011 / 00000111 = 001111 Rest 010 01101011 / 111 = -000 (0, 0 passt nicht) 011 -000 (0, 0 passt nicht) 0110 -000 (0, 0 passt nicht) 01101 - 111 (1, 111 passt) 01100 - 111 (1, 111 passt) 01011 - 111 (1, 111 passt) 01001 - 111 (1, 111 passt) 010 (Rest) Ergebnis: 001111 = 15, Rest 010 = 2

Division mit negativen Zahlen

Vorgehen:
1. Vorzeichen des Resultats voraus bestimmen
2. Rechnung mit positiven Zahlen durchführen
3. Vorzeichen des Resultats und Rests korrigieren

Beispiel: -107 / -7 → beide negativ → Resultat positiv, Rest negativ
Rechne: 107 / 7 = 15 Rest 2 → Ergebnis: 15 Rest -2
